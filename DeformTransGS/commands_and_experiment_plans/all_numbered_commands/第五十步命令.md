# 二、为什么这个 depth bug 特别致命？

透明 mask（透明物体掩码）只能告诉你：

> Gaussian 投影落在透明物体二维区域。

它不能区分：

* 前表面；
* 后表面；
* 玻璃壁另一侧；
* 内部 floaters（漂浮 Gaussian）；
* 同一视线上的多层 Gaussian。

而 TSGS 恰恰专门设计了 first-surface depth（第一表面深度），通过 alpha blending weight（透明度混合权重）的滑动窗口去定位最可能的第一表面；论文也明确把它作为透明表面几何提取组件。([arXiv][1])

所以原来设计：

```text
transparent mask
+
first-surface depth proximity
```

不是装饰。

它的目的就是：

> **从透明物体投影柱里挑出同一表面层附近的 Gaussian。**

现在 depth filter（深度过滤）根本没执行。

那 5972 个 Gaussian 很可能混了多个层。

此时：

[
normal\ p90\approx75^\circ
]

完全可能。

# 我现在对方向的真实评价

说实话，这个结果没有让我觉得“课题死了”。

反而让我觉得我们终于抓到了**真正的 bridge 问题**：

> **PhysGaussian 一类方法可以把 Gaussian 当材料点，是因为 Gaussian 和物理运动学共用离散载体。** ([arXiv][2])
>
> **但 TSGS 是为了透明表面重建设计的表示，它明确把几何学习与外观精修分开，并用 first-surface extraction（第一表面提取）恢复透明表面。** ([arXiv][1])

所以：

> **“TSGS 中每个 Gaussian 的最小轴能不能直接当材料法向”本来就不应该未经验证地假设。**

这可能正是前面 bridge 一直撞墙的原因。

还有，R25 的 geometry–optics conflict（几何—光学冲突）仍然保留，但“几何 opacity 和外观 opacity 解耦”本身已经有近期工作明确提出，尤其透明场景获益，所以不能拿这个当你的主创新。([arXiv][3])

**下一步别救 KIOT，也别杀 KIOT。**

先回答一个更基础的问题：

[
\boxed{
\text{TSGS 里到底有没有可恢复的“材料表面层”？}
}
]

R4 如果是 `DEPTH-NORMAL-BRIDGE-CARRIER`，我认为反而是非常合理的结果：**用 TSGS first-surface geometry 定义材料运动学，用 Gaussian covariance 定义 splat kernel 运动学，再测 KIOT。**

R4 如果真是 `NO-RECOVERABLE-SURFACE-LAYER`，那我就支持你彻底停止 TSGS bridge，不再浪费实验时间。
