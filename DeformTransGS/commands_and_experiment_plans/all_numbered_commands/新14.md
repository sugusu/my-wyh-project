## 一、这次真正证明了什么

相比 N2 只看最大值，现在证据已经强很多：

| 项目               | PAIR-A | PAIR-C |
| ---------------- | -----: | -----: |
| 满足冻结检测条件的比例      | 47.19% | 36.93% |
| 8-bit 量化后不同码值比例  | 40.22% | 24.68% |
| 12-bit 量化后不同码值比例 | 80.93% | 74.21% |
| 覆盖视角             |    6/6 |    6/6 |
| 覆盖通道             |    3/3 |    3/3 |
| 两种表面均成立          |      是 |      是 |

并且 9/9 个冻结参数邻域都保留了响应。

因此可以接受：

> 对于当前微结构驱动的各向异性消光机制，Q1 和 Q2 的信息碰撞会在相当一部分材料点、视角及颜色通道中造成可观察的光学歧义，而不是单个异常样本产生的偶然现象。

PAIR-C 尤其重要：

* Q2 完全相同；
* Q3 不同；
* 局部观察方向完全相同；
* 各向同性 U0 差异约 (10^{-16})；
* 各向异性 U1/U2 产生稳定差异。

这已经是一个合格的**受控反事实证据**。

## 二、为什么“8-bit 鲁棒”还不能完全接受

现在证明的是：

> 量化到 8-bit 后，约 24.7%～40.2% 的成对数值落入了不同码值。

但这不完全等于：

> 在真实 8-bit 观测中对噪声、曝光和量化边界都鲁棒。

因为当前很多 RGB 差异小于：

[
\frac{1}{255}\approx0.00392
]

例如 S0 PAIR-C 的 RGB p95：

[
0.001228
]

只有约 0.31 个 8-bit 码值。

两个数即使相差不到一个码值，也可能恰好位于量化边界两侧，因此变成不同码值。只要改变一点曝光或黑电平，这种现象就可能消失。

而当前噪声实验只报告了 **12-bit**：

```text
sigma=0.0005:
PAIR-A 77.8%
PAIR-C 71.5%

sigma=0.001:
PAIR-A 73.9%
PAIR-C 66.8%
```

没有完成 8-bit 下的噪声、曝光和量化相位测试。

所以现在更稳妥的表述是：

> **在高精度浮点和 12-bit 线性 RGB 中具有明确、分布式的鲁棒信号；在无噪声 8-bit 量化下具有可分性，但完整 8-bit 传感器鲁棒性尚未闭环。**

这不影响论文方向继续。真实相机 RAW 通常本来就是高于 8-bit 的线性数据，后面完全可以将观测协议限定为：

```text
CONTROLLED LINEAR RAW RGB
```

而不是消费级压缩后的 8-bit sRGB 图像。

## 三、81.9865 的处理仍然不合格

虽然正确判断为：

```text
NEAR-ZERO-DENOMINATOR-AMPLIFICATION
```

但输出仍然是：

```text
numerator/denominator:
not_stored ... / inferred
```

原协议要求找到：

* 精确样本；
* 精确分子；
* 精确分母；
* surface；
* microstructure；
* view；
* channel。

现在没有做到，只是推断。

因此形式上：

```text

```

不能完全成立。

正确处理是：

> 重新从原始 confound rows 中计算该指标，准确找到产生 81.9865 的数据行；如果旧文件没有保存，就重算全部 material-only 指标，不能用“推断”代替来源闭环。

这不会推翻主要结果，因为你的主要结论已经依赖 floor-safe metric（带分母下限的安全相对指标）和绝对差异，不再依赖 81.9865。

但科研日志必须修干净。
