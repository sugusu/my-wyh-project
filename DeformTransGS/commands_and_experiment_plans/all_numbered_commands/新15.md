这次我**不直接接受“主线 STOP”**。

不是因为舍不得，而是结果里有一个非常明显的协议异常：

> 无噪声情况下，8/10/12-bit 的量化相位 p10 可分比例分别很高；加入仅 **0.25 LSB** 噪声后，三个位深、PAIR-A 和 PAIR-C 的“最差条件”却全部精确变成 `0.0/0.0`。

这更像是**聚合规则或实现问题**，不能直接解释成物理信号彻底不可观测。

# 一、可以正式接受的结论

## 1. 81.9865 的问题修复完成

这次找到了精确来源：

[
\text{numerator}=3.4098\times10^{-4}
]

[
\text{denominator}=4.1590\times10^{-6}
]

所以：

[
\frac{3.4098\times10^{-4}}{4.1590\times10^{-6}}
=81.9865
]

它确实是**近零分母放大**，不能写成“材料响应变化 8198%”。

修正后的 floor-safe 相对差为：

[
0.7726
]

这一项可正式冻结为：

```text
NEAR-ZERO-DENOMINATOR-AMPLIFICATION
```

旧的 `81.9865` 不得再作为效应强度证据。

## 2. 无噪声量化可分性确实存在

量化相位变化后，PAIR-C 的不同码值比例 p10 为：

* 8-bit：22.82%
* 10-bit：47.36%
* 12-bit：72.78%

这说明信号不是只在某一个偶然量化边界上成立。

PAIR-A 同样保持：

* 8-bit：37.72%
* 10-bit：57.55%
* 12-bit：79.54%

因此不能说：

> 该信号只能存在于浮点数中。

至少在**无噪声的有限位深量化**下，它仍有明显分布支持。

# 二、为什么当前 STOP 结论有问题

原冻结规则要求的是：

> 在所有 exposure、black level、quantization phase 条件形成的分布中，检查 **p10** 的不同码值比例和符号一致率。

但终端输出的是：

```text
worst-condition different-code and sign-consistency
```

也就是取绝对最差组合。

这两个标准不是一回事。

只要某一个曝光、偏置、量化相位组合让大量样本落在同一码值，绝对最差值就可能为 0。它不能代表整体传感器可观测性。

更可疑的是：

```text
8-bit  = 0/0
10-bit = 0/0
12-bit = 0/0
```

三个精度完全一样地归零。

如果问题真的是信号小于量化精度，12-bit 通常不应该和 8-bit 表现得完全相同。常见的实现风险包括：

* 将“量化后相等”的行全部记为符号错误，而不是记为无判定；
* 某个分组为空时默认返回 0；
* 对每个微小分组先取最差，再跨分组继续取最差；
* 噪声 null-control 和 anisotropic effect 使用了不同统计单位；
* 取了单个 seed/phase 的最小值，而非协议要求的跨条件 p10；
* 饱和行或裁剪行被包含进最差统计；
* `sign-consistency` 对真实差值接近零的行没有设置有效性掩码。

因此当前最准确的正式状态是：

```text
METRIC-PROVENANCE: PASS

NO-NOISE QUANTIZED SEPARABILITY: PASS

NOISY SENSOR ROBUSTNESS:
PROTOCOL/AGGREGATION UNRESOLVED

CASE SIGNAL-NOT-ROBUST-AT-12BIT:
NOT YET ACCEPTED
```

# 三、U0 null exceeded = NO 也必须追溯

这一项比全零更重要：

```text
U0 null exceeded by PAIR-C: NO
```

如果含义是：

> 加噪后，PAIR-C 各向异性响应无法显著高于各向同性 U0 的噪声响应，

那确实会阻止 N3。

但现在不知道它是依据：

* 最大值；
* p99；
* 每行比较；
* 分组均值；
* 最差 sensor condition；
* 还是所有条件拼接后的统计。

如果它也是由那个全零“worst condition”触发，就不能独立支持 STOP。
