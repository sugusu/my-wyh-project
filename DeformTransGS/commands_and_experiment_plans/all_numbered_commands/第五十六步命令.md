# 一、现在真正正确的恢复方式已经很清楚

官方 TransLab 的 `transparent_masks` 不是根据 opacity 或材质透明度阈值生成的。

官方脚本首先把所有 mesh 的材质清空并替换成统一的 Diffuse 材质，然后启用 Blender Object Index pass（对象编号通道），最后只取：

```python
id_mask.index = 1
```

对应的对象编号。([GitHub][1])

这意味着一个非常重要的事实：

> **`transparent_masks` 本质上是一个“可见表面属于 pass_index=1 对象吗”的二维对象身份掩码。**

它不是：

> 光学 alpha mask。

也不是：

> 哪些位置看起来透明。

因此，现在恢复 transparent mesh subset（透明网格子集）的正确工具应该是：

> **occlusion-aware labeled z-buffer**
>
> 中文：**考虑遮挡关系的带对象标签深度缓冲。**

简单说：

对完整 `scene_mesh.obj` 实际光栅化。

每个像素不仅存：

[
depth
]

还存当前最近 triangle 的：

* OBJ block id；
* connected component id；
* face id。

然后直接问：

> 当某个 block/component 是相机可见的第一层 mesh 时，它所在像素有多大比例属于 `transparent_masks`？

这和官方 Object Index mask 的语义才真正对应。

官方 README 也明确区分了 `masks/`、`transparent_masks/` 和 `meshes/`：前两者分别是对象掩码和透明对象掩码，mesh 则是导出的场景几何。([GitHub][2])

# 二、为什么不能继续用 surface-sample inside fraction

你现在 top5 中：

```text
Bolt inside-mask = 0.9996
```

这不代表 Bolt 是 transparent object。

原因很简单。

假设一个 mesh 点在三维中位于玻璃瓶后面。

投影以后：

```text
pixel ∈ object mask
```

当然成立。

但真正渲染时，它可能根本不可见。

所以：

[
\text{projected surface point inside mask}
]

不等于：

[
\text{visible surface belongs to mask object}
]

之前的 surface-sample audit 非常适合判断：

> global coordinates 是否完全错位。

它已经完成使命了。

但**不能拿来恢复 object identity（对象身份）**。
