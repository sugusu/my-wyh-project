这次结果是**真正有意义的正结果**，但还没到直接训练 Q1/Q2/Q3 predictor（预测器）的阶段。

## 一、我接受哪些结论

可以正式接受：

```text
CASE MICROSTRUCTURE-ANISOTROPIC-EXTINCTION-ORACLE-READY
```

因为几条最关键的证据都很干净：

* 光学真值生成器没有读取 `Js / lambda / Ct`，非循环性通过；
* 独立重放误差为 0；
* 刚体旋转不改变材料内禀响应，误差约 (10^{-16})；
* 各向同性 U0 在 PAIR-C 中严格不产生方向差异；
* PAIR-A 中 (Q_1) 相同，但各向异性材料产生不同响应；
* PAIR-C 中 (Q_2) 相同，但材料方向不同，产生不同响应；
* 两组对照的局部观察方向完全相同，排除了视角混杂。

所以现在可以说：

> 在这个微结构驱动的各向异性消光机制中，只保存面积标量 (Q_1) 无法区分 PAIR-A；只保存无序主拉伸谱 (Q_2) 无法区分 PAIR-C；材料坐标系中的完整张量状态 (Q_3) 包含区分这些形变所需的信息。

注意范围仍然必须限定为：

```text
CONTROLLED-MICROSTRUCTURE-MECHANISM-ONLY
```

不能推广到所有真实透明材料。

# 二、但当前信号其实比较弱

这是现在最需要警惕的地方。

PAIR-A：

[
\Delta\tau_{\mathrm{rel,max}}=2.27%
]

[
\Delta RGB_{\max}=0.001387
]

PAIR-C：

[
\Delta\tau_{\mathrm{rel,max}}=2.46%
]

[
\Delta RGB_{\max}=0.001228
]

它们确实超过了冻结阈值：

```text
tau relative >= 2%
RGB absolute >= 0.001
```

但只是略微超过。

更重要的是：

[
\frac{1}{255}\approx0.00392
]

也就是说，当前最大 RGB 差异甚至小于普通 8-bit 图像的一个灰度码值。

换算一下：

* 8-bit：约 0.31～0.35 个码值；
* 10-bit：约 1.26～1.42 个码值；
* 12-bit：约 5.0～5.7 个码值。

因此，目前更准确的表述应该是：

> 该机制在浮点或高位深线性 RGB 中具有可检测差异，但尚未证明在普通 8-bit 图像和现实噪声下稳定可见。

所以 N1 中的：

```text
ORDINARY_RGB_TRANSMITTED_INTENSITY
```

需要进一步限定为：

```text
LINEAR HIGH-BIT-DEPTH RGB
WITH CONTROLLED ILLUMINATION
```

至少现阶段如此。

# 三、81.9865 这个结果不能当强证据

这一项：

```text
PAIR-C material-only tau_eff relative difference:
81.98654863326676
```

与总体 tau 差异的 2.455% 完全不在一个数量级。

最可能的情况是：

[
\Delta\tau_{\mathrm{material}}
]

本身接近零，计算相对误差时分母过小，导致比值被放大。

所以在没有检查以下内容前：

* 分子绝对值；
* 分母分布；
* median/p90/p99；
* 是否存在近零分母；

这项结果必须标记为：

```text
UNSTABLE-RELATIVE-METRIC
```

不能写成“材料响应变化达到 81.99%”。

# 四、现在不能直接做 N3 的另一个原因

目前报告主要使用：

```text
maximum difference
```

最大值。

但 486 万行数据里，只要一个：

* 特定表面点；
* 特定通道；
* 特定观察方向；
* 特定材料；

超过阈值，Gate 就会通过。

这只证明：

> 该机制存在至少一个可区分例子。

尚未证明：

> 该机制在一个稳定、非稀疏的区域内可观察。

因此下一步不应该马上训练 predictor。

否则可能出现：

```text
Q3 test error < Q2 test error
```

但其实只是模型学习了极少数弱信号，论文结论很脆。
