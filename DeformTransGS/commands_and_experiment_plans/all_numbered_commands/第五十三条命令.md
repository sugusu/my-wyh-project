# 三、这不等于课题死亡

真正被杀掉的是：

> **“直接把 TSGS 中每个 Gaussian 当作一个有稳定材料身份、材料法向和形变梯度的点。”**

没有被杀掉的是：

> **“透明 Gaussian 在可靠表面运动学载体下，需要随局部面积变化传输光学状态。”**

因此，下一条合理路线不是再修 TSGS point cloud，而是引入：

> **explicit surface scaffold，显式表面支架。**

例如 mesh（网格）提供：

* 稳定材料身份；
* 邻接拓扑；
* 表面法向；
* 局部形变梯度 (F)；
* 面积伸长率 (J_s)。

Gaussian 只负责：

* 外观和透明渲染；
* 随 mesh 传输中心与 covariance；
* 用 KIOT 或其他规则更新 optical opacity。

把 Gaussian 绑定到 mesh、cage 或其他显式代理上进行形变本身已经有大量工作，因此**mesh scaffold 不能当创新点**。Mesh-based GS、MaGS、UniMGS 等都已使用显式 mesh 组织或驱动 Gaussian deformation。([arXiv][2])

你的创新只能继续放在：

> **面积形变下，透明 Gaussian 的光学状态如何传输。**

# 四、下一条主线应该怎样改写

旧主线：

> TSGS Gaussian 点云 → 直接视作材料点 → 形变 → KIOT。

这条已经终止。

新主线：

> **透明重建外观 Gaussian + 显式表面运动学 scaffold → 形变 → renderer-aware optical-state transport。**

更具体地说：

1. 用透明重建方法获得 canonical Gaussian appearance；
2. 使用 GT mesh 或重建 mesh 作为表面支架；
3. 将 Gaussian 绑定到 mesh face；
4. 由 mesh deformation 得到真实 (F)、normal 和 (J_s)；
5. 统一传输 Gaussian 中心与 covariance；
6. 比较固定 opacity、(\tau/J_s)、(q,o)、KIOT；
7. 使用物理渲染产生的 deformed GT 做最终评价。

PhysGaussian证明了使用连续介质形变梯度更新 Gaussian kernel 是合理的运动学路线，但它并不解决透明材料面积变化时的 optical-state transport。([arXiv][3])
