这次我**不接受**最终结论：

> `CASE INSTALLED-RASTERIZER-NOT-ATTRIBUTE-DIFFERENTIABLE`

正确状态应该是：

> **`RASTERIZER-RUNTIME-ENVIRONMENT-UNRESOLVED`**
>
> 中文：**光栅化器运行环境尚未闭环。**

原因非常简单：

```text
installed rasterizer module/path: UNAVAILABLE
```

并且失败位置是：

```python
from . import _C
```

这表示 **Python 根本没有成功加载 CUDA/C++ 扩展 `_C`**。

所以实际实验链是：

[
\text{import failed}
\rightarrow
\text{forward not run}
\rightarrow
\text{backward not run}
]

而不是：

[
\text{forward works}
\rightarrow
\text{backward grad=0}
]

因此：

> **目前没有任何证据证明 rasterizer 不可微。**

官方 TSGS 安装说明明确要求单独执行 `pip install submodules/diff-first-surface-rasterization`；该子模块的 `setup.py` 会编译名为 `diff_first_surface_rasterization._C` 的 CUDAExtension。 ([GitHub][1])

更关键的是，官方 Python autograd wrapper 的 `backward()` 实际调用 `_C.rasterize_gaussians_backward`，并返回 `grad_colors_precomp`、`grad_opacities` 和 `grad_sh`；官方 TSGS 训练代码也通过 `gaussian_renderer.render()` 得到图像后执行 `loss.backward()`。 ([GitHub][2])

所以现在最应该怀疑的是：

> **Codex 执行 Stage 4 的 Python 环境，不是当初真正跑通 TSGS baseline 的 Python/torch/CUDA extension 环境。**

很可能就是经典的：

```text
TSGS training env A
    _C installed and loadable

Codex current env B
    Python can see package __init__.py
    but compiled _C missing / incompatible
```

甚至可能：

```text
package path 来自 source checkout
_C.so 来自另一个 conda env
```

所以现在**不要修计算图**。

先找到“当初真正跑通 TSGS 的环境”。
