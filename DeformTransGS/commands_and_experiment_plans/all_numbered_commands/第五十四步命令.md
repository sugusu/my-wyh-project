这次终于是一个**比较干净的 protocol mistake（协议错误）**。

不是 KIOT 理论又塌了。

不是 TSGS 又出问题了。

也不是 mesh scaffold 已经失败。

而是我们误把：

[
\boxed{\text{scene mesh}}
]

当成了：

[
\boxed{\text{transparent object mesh}}
]

官方源码已经明确说明前者导出的是场景中所有可见 mesh，而透明掩码只选择 object index 1。([GitHub][1])

所以这一步别再扩大实验。

**先跑 R1。**

判断非常简单：

* `scene_mesh ↔ masks` 高 IoU，同时 `pass_index=1 mesh ↔ transparent_masks` 高 IoU：**直接重跑 3.6A 生死门。**
* 整场景 mesh 连 `masks` 都对不上：查坐标链。
* 整场景对得上，但透明 mesh 根本恢复不出来：**停止 GT mesh scaffold 路线。**

这次 KIOT 状态仍然是：

> **UNDECIDED。**
