```text

```

分别对以下阈值报告类别比例：

```text
方案 A：
correct < 0.005
wrong >= 0.020

方案 B：
correct < 0.010
wrong >= 0.030

方案 C：
correct < 0.005
wrong >= 0.010
```

注意：

* 这一步只检查标签敏感性；
* 不允许选取“让 AUPRC 最好”的阈值；
* 如果 wrong 比例低于 1% 或高于 80%，当前二分类标签不适合直接评价；
* 此时优先使用连续 Spearman、风险分位数误差曲线，而不是硬二分类。

输出：


# 十二、分开评价两类几何错误

当前组合风险可能失败，是因为“背景漂浮”和“内部错误”机制不同。

创建：

```text
src/reliability/evaluate_typed_reliability.py
```

第一版定义两个分析分支：

## A. 背景漂浮风险

候选风险：

```text
R_background =
0.55 × E_mask
+ 0.25 × E_support
+ 0.20 × E_scale
```

仅作诊断，不修改正式配置。

评价目标：

```text
低 mask support
且 GT mesh distance 大
```

## B. 内部/错误深度风险

候选风险：

```text
R_internal =
0.40 × E_normal
+ 0.35 × E_depth
+ 0.25 × E_support
```

只在：

```text
mask support >= 0.7
```

的 Gaussian 上评价。

输出：

```text
outputs/debug/stage1b_scene01/typed_reliability_metrics.json
outputs/debug/stage1b_scene01/background_risk_colored.ply
outputs/debug/stage1b_scene01/internal_risk_colored.ply
outputs/debug/stage1b_scene01/typed_reliability_report.md
```

注意：

