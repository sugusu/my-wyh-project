# 六、实现跨视角 Mask Consistency

创建：

```text
src/features/extract_mask_consistency_v2.py
```

对 candidate-object Gaussian 计算以下原始量。

## 6.1 有效视图

只有以下条件全部满足才算有效视图：

```text
Gaussian 投影在图像内；
Gaussian 深度为正；
Gaussian 在该视图可见；
对应 mask 文件有效；
不是无效或背景填充值。
```

无效视图必须排除，不能填 0。

---

## 6.2 Mask 支持均值

```text
S_mask =
加权平均 transparent_mask 投影值
```

风险：

```text
E_mask_mean = 1 - S_mask
```

---

## 6.3 跨视角 Mask 冲突

令每个有效视图的 mask 命中为：

```text
m_iv ∈ [0,1]
```

计算：

```text
V_mask = weighted variance(m_iv)
```

将其分位数归一化：

```text
E_mask_variance =
percentile_normalize(V_mask)
```

它用于检测：

```text
某个 Gaussian 在部分视图位于透明物体内，
但在其他视图投影到透明 mask 外。
```

---

## 6.4 Mask 边界风险

计算投影点到 mask 边界的有符号距离。

定义：

```text
负值：mask 外
正值：mask 内
```

边界带：

```text
|distance| <= 3 pixels
```

保存：

```text
boundary_view_ratio
outside_view_ratio
inside_view_ratio
```

第一版边界风险：

```text
E_mask_boundary =
0.7 × outside_view_ratio
+ 0.3 × boundary_view_ratio
```

该公式固定，不使用 GT 调节。

---

## 6.5 Mask 跨视角风险

分别保存，不要立刻只保留组合：

```text
E_mask_mean
E_mask_variance
E_mask_boundary
```

诊断组合：

```text
E_mask_cv =
0.40 × E_mask_mean
+ 0.30 × E_mask_variance
+ 0.30 × E_mask_boundary
```

输出：

```text
outputs/reliability/<scene>/iter_15000/mask_valid_view_count.npy
outputs/reliability/<scene>/iter_15000/mask_mean_support.npy
outputs/reliability/<scene>/iter_15000/mask_variance.npy
outputs/reliability/<scene>/iter_15000/mask_boundary_ratio.npy
outputs/reliability/<scene>/iter_15000/mask_risk_mean.npy
outputs/reliability/<scene>/iter_15000/mask_risk_variance.npy
outputs/reliability/<scene>/iter_15000/mask_risk_boundary.npy
outputs/reliability/<scene>/iter_15000/mask_risk_cv.npy
outputs/reliability/<scene>/iter_15000/mask_consistency_stats.json
```

# 七、实现局部 PCA 法线

当前 Gaussian 最短轴法线覆盖或稳定性不足，因此不要继续只依赖 Gaussian rotation。

创建：

```text
src/features/extract_pca_normal_conflict.py
```

## 7.1 局部 PCA 法线

对 candidate-object Gaussian 建立 kNN：

```text
k = 24
```

计算邻域协方差矩阵：

```text
C_i =
sum_j w_ij
(x_j - mean_i)(x_j - mean_i)^T
```

最小特征值对应的特征向量为：

```text
n_pca
```

PCA 法线正负不重要，后续使用绝对 cosine。

---

## 7.2 Planarity confidence

特征值：

```text
lambda_0 <= lambda_1 <= lambda_2
```

定义：

```text
planarity_confidence =
(lambda_1 - lambda_0) / (lambda_2 + eps)
```

必须保存，不使用 GT 选阈值。

只有：

```text
邻居数 >= 8
planarity_confidence >= 0.10
```

时 PCA 法线有效。

---

## 7.3 跨视角法线冲突

将 `n_pca` 转换到相机坐标系，与 TSGS normal prior 比较：

```text
e_normal_iv =
1 - abs(dot(n_pca_camera, normal_prior))
```

仅统计：

```text
投影有效；
位于 transparent mask 内；
normal prior 有效；
该 Gaussian 在该视图可见。
```

跨视角聚合分别保存：

```text
median_normal_error
mean_normal_error
normal_error_variance
valid_normal_view_count
```

第一版主风险：

```text
E_normal_pca =
weighted median(e_normal_iv)
```

不要使用 GT 修改公式。

输出：

```text
outputs/reliability/<scene>/iter_15000/pca_normal.npy
outputs/reliability/<scene>/iter_15000/pca_planarity.npy
outputs/reliability/<scene>/iter_15000/pca_normal_valid.npy
outputs/reliability/<scene>/iter_15000/pca_normal_conflict.npy
outputs/reliability/<scene>/iter_15000/pca_normal_variance.npy
outputs/reliability/<scene>/iter_15000/pca_normal_stats.json
```

# 十一、只有单项验证后才组合风险

如果至少两个特征在两个场景方向一致，其中至少一个是真正跨视角特征，才创建组合风险。

建议预注册组合：

```text
R_stage1h =
0.45 × E_mask_cv
+ 0.35 × E_normal_pca
+ 0.20 × E_scale_v2
```

注意：

* 只有对应特征有效时才加入；
* 无效特征不得填 0；
* 缺失特征时重新归一化剩余权重；
* 不通过 GT 网格搜索这些权重。

如果只有一个特征有效：

```text
不要构建组合风险。
```

## PASS

```text
2. 至少两个特征方向跨场景一致；
```

PASS 后：

```text
允许进入 Stage 2 prune-only。
```

## CONDITIONAL

```text
只有一个跨视角特征方向一致，
但强度不足或另一个特征仅为弱信号。
```

处理：

```text
可增加 scene_07 验证，但仍不进入 prune。
```

## FAIL

满足任意情况：

```text
mask 和 PCA normal 均跨场景方向不一致；
只有 E_scale_v2 有效；
所有跨视角特征 CI 包含 0；
组合风险跨场景方向相反。
```

处理：

```text
停止当前 Gaussian-level reliability 路线。
不得进入 Stage 2。
```
