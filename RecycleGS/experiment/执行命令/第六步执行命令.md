# 三、本阶段要回答的问题

必须验证以下假设：

```text
H1：高风险 Gaussian 是否真的更偏离 GT 表面？
```

风险特征包括：

```text
E_mask       mask 排除风险
E_normal     跨视角法线冲突
E_depth      深度排序冲突
E_support    局部表面支持不足
E_scale      scale 异常
S_contrib    渲染贡献，仅作安全门，不进入风险总分
```

第一版风险分数：

```text
R =
0.15 × E_mask
+ 0.30 × E_normal
+ 0.20 × E_depth
+ 0.25 × E_support
+ 0.10 × E_scale
```

注意：

1. `S_contrib` 不加入风险总分；
2. GT mesh distance 只能用于最后评价；
3. first-surface depth 只作为冻结 checkpoint 的参考排序；
4. 当前阶段不得删除或改变任何 Gaussian。

# 九、实现 Mask 排除风险

创建：

```text
src/features/extract_mask_support.py
```

对每个 Gaussian 投影到 64 个分析视图。

第一版权重优先顺序：

1. renderer 提供的真实 per-Gaussian visibility/contribution；
2. visibility_filter × projected radius × opacity；
3. valid projection × projected radius × opacity。

若使用第 2 或第 3 种，必须在输出报告中标记：

```text
contribution_mode = proxy
```

计算：

```text
S_mask = 加权 mask 支持
E_mask = 1 - S_mask
```

注意：

* 透明 mask 外投影增加风险；
* mask 内投影不能证明几何正确；
* 完全不可见的 Gaussian 不参与该视图统计；
* 所有投影必须分块处理，不能一次创建 118557 × 64 的巨型中间张量。

输出：

```text
outputs/reliability/scene_01/mask_support.npy
outputs/reliability/scene_01/mask_risk.npy
```

# 十一、实现深度排序冲突

创建：

```text
src/features/extract_depth_order_conflict.py
```

重要原则：

```text
不得把靠近 depth_ref 当作正确；
只惩罚明显位于冻结参考第一表面后方，
同时仍具有有效投影或贡献的 Gaussian。
```

定义：

```text
tau_abs = absolute_tolerance_scale × 当前视图中位 Gaussian 法向尺度
tau_rel = relative_tolerance × depth_ref

tau = max(tau_abs, tau_rel)
```

单视图风险：

```text
如果 z_gaussian > depth_ref + tau：
    e_depth = (z_gaussian - depth_ref) / (depth_ref + epsilon)
否则：
    e_depth = 0
```

跨视角使用加权中位数。

输出：

```text
outputs/reliability/scene_01/depth_order_conflict.npy
outputs/reliability/scene_01/depth_order_stats.json
```

报告中必须注明：

```text
depth_ref 仅用于相对排序参考，不是真实几何监督。
```

# 十二、实现局部表面支持

创建：

```text
src/features/extract_surface_support.py
```

要求：

1. 使用 Gaussian xyz 建立 kNN；
2. 默认 `k=24`；
3. 使用分块或 FAISS / scipy cKDTree；
4. 不允许构造完整 N×N 距离矩阵；
5. 邻域半径基于局部最近邻尺度；
6. 计算位置邻近与法线一致性的联合支持。

计算：

```text
S_surface =
mean[
    exp(-distance² / (2 sigma_p²))
    × abs(dot(n_i, n_j))
]
```

归一化后：

```text
E_support = 1 - normalized(S_surface)
```

输出：

```text
outputs/reliability/scene_01/surface_support.npy
outputs/reliability/scene_01/surface_support_risk.npy
outputs/reliability/scene_01/surface_support_stats.json
```

# 十八、Gate 1A 判断规则

scene_01 只能称为：

```text
Gate 1A：单场景初步验证
```

不能写“Gate 1 已通过”。

Gate 1A 建议满足：

```text
1. 总风险 AUPRC 明显高于 wrong 类别比例；
2. top-10% 风险组错误率至少为整体错误率 1.5 倍；
3. top-10% 风险组 mesh distance 显著高于 bottom-10%；
4. 至少两个特征与 GT 误差具有稳定正相关；
5. 坐标系、投影和 normal convention 检查通过。
```

正式 Gate 1 需要：

```text
scene_01 + scene_03 趋势一致
```

如果 Gate 1A 失败：

1. 不得进入 prune；
2. 首先检查坐标和 normal convention；
3. 检查 mask 文件与相机图像名称是否对应；
4. 检查 depth_ref 方向和单位；
5. 分开分析背景漂浮点与内部错误点；
6. 仅在确认实现无误后再调整风险权重。
