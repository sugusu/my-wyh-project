# 五、Schedule Control 的评价语义必须修正

`schedule_control`：

```text

→ 不剪枝
→ 正常无densification训练500步
→ control 15.5k
```

因此：

```text

prune_psnr = N/A
rec_psnr = schedule_control 15.5k PSNR
```

对 schedule_control：

```text
C1 Recovery improves over immediate prune = N/A
```

不能判 FAIL。

Schedule control 只评价：

```text
C2:
control_15500 PSNR 与 baseline_15000 的关系
```

它主要用于判断：

> 普通多训练500步自身会带来多少变化。

# 十一、Stage 2B Gate 正确规则

## C1：Recovery 相比 immediate prune

仅对四种剪枝方法：

```text
random
low_opacity
low_contribution
mask_risk
```

评价：

```text
recovery PSNR >= immediate prune PSNR
或
recovery PSNR drop 已显著恢复
```

同时报告：

```text
delta_psnr_recovery =
PSNR_recovery - PSNR_prune
```

C1 不适用于 schedule_control。

---

## C2：Schedule control

计算：

```text
delta_control =
PSNR_control_15500 - PSNR_baseline_15000
```

这里只作为训练 schedule 参考。

不要强制：

```text
control_15500 >= baseline_15000
```

因为额外训练500步并不保证 test PSNR 单调增长。

因此 C2 改为：

```text
schedule control metric successfully measured
```

并记录 delta。

---

## C3：Mask-risk 与 Random

两个场景分别要求：

```text
mask_risk final PSNR >= random final PSNR - 0.01
mask_risk final SSIM >= random final SSIM - 0.0005
mask_risk final LPIPS <= random final LPIPS + 0.001
```

---

## C4：数量稳定

保持现有规则：

```text
schedule_control:
N_final = N_baseline

pruned:
N_final = N_baseline - K
```

---

## C5：删除对象质量

继续使用 Stage 2A.5 已验证结果：

```text
mask-risk removed mean d_center_norm
>
random removed mean d_center_norm
```

两个场景都必须成立。

---

## C6：Geometry Proxy

读取恢复后的：

```text
SPR
d_shift
```

要求：

```text
mask_risk SPR >= random SPR
```

`d_shift` 必须先明确“越小越好还是越负越好”。

不得只写：

```text
mask_risk d_shift <= random
```

必须在代码中定义该 proxy 的物理语义。

如果当前 recovery geometry proxy 尚未生成：

```text
C6 = PENDING
```

最终最多输出：

```text
CONDITIONAL PASS
```

不能输出完整 PASS。
