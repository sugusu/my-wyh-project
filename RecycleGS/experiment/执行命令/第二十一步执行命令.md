# 八、State Pairing 判断规则

## Case A：辅助模型错配确认

满足：

```text

G155+A15 ≈ 17.54

G155+A155 恢复至接近 G15+A15
```

判定：

```text
ROOT_CAUSE =
AUXILIARY_MODEL_STATE_MISMATCH
```

说明：

> Recovery evaluator paired iteration-15500 Gaussian appearance parameters with stale iteration-15000 AppModel/SpecularModel weights.

---

## Case B：AppModel 自身导致退化

满足：

```text
G15+A15 ≈ 22.39

G15+A155 明显下降

G155+A155 仍明显下降
```

判定：

```text
ROOT_CAUSE =
APPMODEL_RECOVERY_OPTIMIZATION_DRIFT
```

此时检查 custom recovery 的 AppModel training loop 是否与 TSGS 官方完全一致。

---

## Case C：Gaussian appearance 参数导致退化

满足：

```text
G155+A15 下降
G155+A155 同样下降
G15+A155 正常
```

判定：

```text
ROOT_CAUSE =
GAUSSIAN_APPEARANCE_OPTIMIZATION_DRIFT
```

重点检查：

```text
f_dc
f_rest
transparency
```

训练 loss 和 optimizer state。

---

## Case D：所有 matching state 都下降

满足：

```text
G155+A155 ≈17.5
```

且 state bundle 确认完整。

此时：

```text
完整 recovery continuation 本身确实导致统一 test PSNR下降
```

进入 Native Resume Control。
