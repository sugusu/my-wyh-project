## 4.1 Spearman 必须保留符号

禁止：

```python
best_spearman = max(abs(rho) for rho in correlations)
```

改为同时输出：

```python
best_positive_spearman
strongest_absolute_spearman
strongest_absolute_spearman_signed
```

例如：

```json
{
  "best_positive_spearman": 0.173,
  "strongest_absolute_spearman": 0.3417,
  "strongest_absolute_spearman_signed": -0.3417
}
```

报告解释规则：

```text
rho > 0：
风险方向与几何误差一致。

rho < 0：
风险方向相反，可能是特征定义、归一化或语义错误。

abs(rho) 大但 rho 为负：
不能解释成风险有效。
```

# 六、诊断 Surface Support 风险方向

创建：

```text
src/features/diagnose_surface_support_semantics.py
```

读取 candidate-object domain，并同时评价以下原始量：

```text
S_position
S_normal
S_combined
E_support_saved
1 - percentile_normalize(S_combined)
percentile_normalize(S_combined)
```

对每个量报告：

```text
Spearman with d_center_norm
Spearman with d_surface_proxy_alpha1
Spearman with d_surface_proxy_alpha2
top10 mean error
bottom10 mean error
top10/bottom10 ratio
```

必须检查：

```text
E_support_saved
```

是否真的等于：

```text
1 - percentile_normalize(S_combined)
```

计算最大绝对误差：

```text
max_abs_difference
```

如果大于：

```text
1e-5
```

说明保存或归一化实现有错误。

输出：

```text
outputs/debug/stage1e_scene01/support_semantics.json
outputs/debug/stage1e_scene01/support_semantics.csv
outputs/debug/stage1e_scene01/support_semantics_report.md
```

判断：

### 情况 A：实现反了

如果：

```text
S_combined 与误差负相关
```

而：

```text
E_support_saved 也与误差负相关
```

说明风险转换实现错误。

### 情况 B：语义不成立

如果：

```text
S_combined 与误差正相关
E_support 与误差负相关
```

说明在透明物体 candidate domain 中：

```text
局部支持高 ≠ 几何可靠
```

此时 surface support 暂时不能作为正向风险特征。

不得仅为了指标变好而对其乘以 `-1`。
