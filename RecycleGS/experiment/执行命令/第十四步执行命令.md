# 五、冻结 Type-A 风险定义

创建：

```text
configs/stage2/type_a_prune_locked.yaml
```

内容：

```yaml
checkpoint_iteration: 15000

risk_type: mask_inconsistent_floater

candidate_domain:
  min_valid_views: 5
  min_mask_support: 0.20
  strong_background_threshold: 0.05

primary_risk:
  name: mask_risk_mean
  definition: 1_minus_cross_view_mask_support

ambiguity_protection:
  enabled: true
  max_mask_variance: 0.20
  max_boundary_view_ratio: 0.50

contribution_protection:
  enabled: true
  max_contribution_percentile: 0.80

disabled_features:
  - E_support_v2
  - pca_normal_conflict
  - depth_order_conflict
  - mask_risk_variance

analysis_only_features:
  - E_scale_v2
  - mask_risk_boundary
  - mask_risk_cv

prune:
  global_ratio: 0.005
  max_eligible_fraction: 0.30
  seed: 0
```

物理解释：

```text
mask_risk_mean 高：
Gaussian 在多个独立视角中缺乏透明物体 mask 支持。

mask variance 高：
可能位于真实轮廓或薄边区域，视为 ambiguous，暂不删除。

boundary ratio 高：
可能是杯口或轮廓 Gaussian，暂不删除。

contribution 高：
即使风险高，也先保护，避免明显破坏渲染。
```

# 七、确定删除数量

由于当前风险只覆盖 Type-A mask 不一致 Gaussian，不能直接删除总 Gaussian 的 2%、5%、10%。

第一轮固定：

```text
K_global = round(0.5% × N_total)
K_pool = round(30% × N_eligible)

K = min(K_global, K_pool)
```

即：

```text
全局删除比例最多 0.5%
候选池最多删除 30%
```

必须保存：

```text
N_total
N_eligible
K_global
K_pool
K_final
actual_global_ratio
actual_eligible_ratio
```

如果：

```text
K < 100
```

则停止并报告 eligible pool 太小。

后续只有 0.5% 删除安全后，才允许测试 1.0%。

不要立即测试 2%、5%、10%。

## 2. Low-opacity

```text

```

## 3. Low-contribution

```text
eligible pool 中 contribution 最低的 K 个
```

## 4. Mask-risk prune

```text
eligible pool 中 mask_risk_mean 最高的 K 个
```

## 5. Oracle

```text
eligible pool 中 GT mesh distance 最高的 K 个
```

Oracle 仅用于离线上界分析：

* 不参与方法；
* 不参与阈值；
* 不参与实际决策；
* 论文中明确标记为 oracle。


建议工程判断：

```text
removed mean distance >= 1.25 × random
PSNR drop <= random PSNR drop
Chamfer increase <= 2%
F-score decrease <= 1 个百分点
```

这些是工程 Gate，不是论文硬阈值。

如果失败：

