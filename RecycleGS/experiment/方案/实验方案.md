# RecycleGS 修正版完整实验方案

下面这版保留了你原方案中“可靠性检测—预算回收—欠覆盖重播种—局部精修”的主线，同时修正三个关键问题：

1. 不再把 TSGS 的 first-surface depth 当作真实几何；
2. 不再把低光学 opacity 直接等同于几何欠覆盖；
3. 固定预算作为公平约束，而不是唯一创新点。

原方案的总体方向和分阶段 Gate 设计是正确的。

---

# 一、研究定位

## 1.1 暂定题目

> **RecycleGS: Cross-View Geometry-Reliability-Guided Gaussian Recycling for Transparent Surface Reconstruction**

中文：

> **跨视角几何可靠性引导的透明表面 Gaussian 回收与重分配**

这里加入 `Cross-View`，是为了突出你的方法不是依靠单视图渲染误差或模型自身深度进行循环判断。

---

## 1.2 研究边界

本项目聚焦：

* 具有主导第一表面的透明物体；
* 外部透明表面的几何恢复；
* TransLab 类型的杯子、瓶子、玻璃器皿；
* 基于多视图 RGB、透明 mask 和法线先验的重建。

第一版不处理：

* 玻璃容器中的液体；
* 多层嵌套透明物；
* 同一像素多个有效几何表面；
* 复杂多次折射；
* 显式折射率估计；
* 动态透明物体。

论文中应明确这一范围，避免方法被要求解决所有透明介质问题。

---

## 1.3 核心科学问题

透明场景中的高梯度和高渲染贡献，不一定对应真实物体表面，可能来自：

* 折射后的背景纹理；
* 镜面高光；
* 前后表面混叠；
* 透明物体内部；
* 错误深度层；
* mask 边缘伪结构。

标准 densification 会在高梯度位置继续 split 或 clone，可能将更多 Gaussian 分配到错误区域。

本项目研究：

> 能否在不依赖测试场景真实几何的情况下，通过跨视角几何一致性识别不可靠 Gaussian，并将有限表示预算从错误区域转移到经过多视角验证的欠覆盖透明表面？

---

## 1.4 核心假设

需要依次验证四个假设：

### H1：检测假设

高几何风险 Gaussian 与真实表面误差显著相关。

### H2：删除假设

在相同删除比例下，可靠性引导删除比 random、low-opacity 和 low-contribution 更安全。

### H3：重分配假设

将回收预算重新播种到跨视角验证的欠覆盖表面，优于随机重生和普通 densification。

### H4：固定预算假设

在 Gaussian 总量和总训练步数接近的条件下，RecycleGS 获得更好的几何质量。

必须按照 H1 → H2 → H3 → H4 顺序验证。

---

# 二、方法总体流程

```text
TSGS训练至中期
        ↓
冻结当前模型和参考几何图
        ↓
提取每个Gaussian的跨视角几何特征
        ↓
计算风险并划分：
retain / recycle / ambiguous
        ↓
执行prune-only安全性验证
        ↓
检测透明表面的几何欠覆盖区域
        ↓
由源视图生成3D候选点
        ↓
使用其他视图进行跨视角验证
        ↓
删除K个Gaussian，并生成K个新Gaussian
        ↓
先几何、后外观的局部精修
        ↓
恢复全局训练至固定总迭代数
```

总数量满足：

[
N_{\text{after}}=N_{\text{before}}
]

正式比较时：

[
\frac{|N_{\text{ours}}-N_{\text{baseline}}|}
{N_{\text{baseline}}}<1%
]

# 六、Stage 1：几何可靠性估计

这是整个项目最关键的部分。

此阶段只分析，不删除，也不重新生成。

---

## 6.1 冻结参考状态

在 15k checkpoint：

1. 冻结全部 Gaussian；
2. 冻结相机；
3. 渲染参考 depth、normal 和几何支持图；
4. 后续 reliability 计算全部使用该冻结状态。

不能边计算可靠性边改变模型。

---

## 6.2 每个 Gaussian 的基本特征

对 Gaussian (g_i) 提取：

[
x_i=
[
E_i^{mask},
E_i^{normal},
E_i^{depth},
E_i^{support},
E_i^{scale},
S_i^{contrib}
]
]

其中：

* (E) 越大表示风险越高；
* (S_i^{contrib}) 不直接作为风险，而作为删除安全门。

---

## 6.3 Mask 排除风险

定义 Gaussian 在可见视角中的加权 mask 支持：

[
S_i^{mask}
==========

\frac{
\sum_{v\in\mathcal V_i}
w_{iv}M_v(\pi_v(\mu_i))
}{
\sum_{v\in\mathcal V_i}w_{iv}+\epsilon
}
]

对应风险：

[
E_i^{mask}=1-S_i^{mask}
]

这里必须注意：

* mask support 低，可以说明 Gaussian 很可能属于背景；
* mask support 高，不能证明 Gaussian 位于真实表面。

因此 mask 主要用于排除背景漂浮点，不是内部错误点的主要判据。

---

## 6.4 法线冲突

Gaussian 法线由最短尺度轴确定：

[
n_i=R_i e_{\min}
]

投影到视角 (v) 后：

[
e_{iv}^{normal}
===============

1-
\left|
n_{iv}^{\top}N_v^{prior}(\pi_v(\mu_i))
\right|
]

使用加权中位数：

[
E_i^{normal}
============

\operatorname{WeightedMedian}*{v}
e*{iv}^{normal}
]

同时保存：

[
V_i^{normal}
============

\operatorname{WeightedVar}*{v}
e*{iv}^{normal}
]

第一版只使用 (E_i^{normal})，方差只用于分析。

---

## 6.5 跨视角深度排序冲突

不能简单把当前 Gaussian 与当前模型自己的 first-surface depth 比较后直接判错。

第一版使用更保守的“深度排序冲突”：

[
e_{iv}^{depth}
==============

\begin{cases}
\frac{z_{iv}-D_v^{ref}(p)}{D_v^{ref}(p)+\epsilon},
&z_{iv}>D_v^{ref}(p)+\tau_d\
0,&\text{otherwise}
\end{cases}
]

其中：

* (D_v^{ref}) 是冻结 checkpoint 的参考表面深度；
* 只惩罚明显位于参考第一表面后方、却仍产生贡献的 Gaussian；
* 不把“与参考深度接近”直接解释为正确。

聚合：

[
E_i^{depth}
===========

\operatorname{WeightedMedian}*{v}
e*{iv}^{depth}
]

深度项只能作为辅助项，不能独立决定回收。

---

## 6.6 局部表面支持度

对于 Gaussian (g_i)，在其邻域中寻找位置和法线相近的 Gaussian：

[
\mathcal N_i=
\left{
g_j:
|\mu_j-\mu_i|<r_i
\right}
]

定义邻域表面支持：

[
S_i^{surface}
=============

\frac{1}{|\mathcal N_i|}
\sum_{j\in\mathcal N_i}
\exp
\left(
-\frac{|\mu_i-\mu_j|^2}{2\sigma_p^2}
\right)
\left|
n_i^\top n_j
\right|
]

对应风险：

[
E_i^{support}=1-\operatorname{Normalize}(S_i^{surface})
]

孤立、漂浮或错误深度层 Gaussian 通常缺少稳定的局部表面邻域。

---

## 6.7 Scale 异常

记录：

[
r_i^{scale}
===========

\frac{s_{i,\max}}{s_{i,\min}+\epsilon}
]

以及：

[
v_i^{scale}
===========

s_{ix}s_{iy}s_{iz}
]

将 scale 转换为分位数风险：

[
E_i^{scale}
===========

\max
\left(
P(r_i^{scale}),
P(v_i^{scale})
\right)
]

其中 (P) 表示在当前场景 Gaussian 分布中的异常分位数。

Scale 只能是辅助项，不能单独用于删除。

---

## 6.8 渲染贡献

定义累计贡献：

[
S_i^{contrib}
=============

\sum_{v,p}w_{iv}(p)
]

工程实现可以使用：

* 累计 alpha contribution；
* 贡献像素数；
* 渲染 loss 对 Gaussian 的梯度；
* 每个 Gaussian 被前向访问的次数。

Contribution 不直接参与风险总分，而作为删除安全门：

```text
高风险 + 低或中等贡献 → 可以回收
高风险 + 高贡献       → ambiguous，暂不删除
```

---

## 6.9 风险组合

第一版使用可解释线性组合：

[
R_i=
\lambda_m E_i^{mask}
+
\lambda_n E_i^{normal}
+
\lambda_d E_i^{depth}
+
\lambda_u E_i^{support}
+
\lambda_s E_i^{scale}
]

初始权重：

```yaml
mask: 0.15
normal: 0.30
depth_order: 0.20
surface_support: 0.25
scale: 0.10
```

只在 scene_01 和 scene_03 上调整。

不要第一版就训练 MLP 或 uncertainty 网络。

# 七、Stage 1A：可靠性有效性评价

## 7.1 GT 只用于离线评价

GT mesh 只能用于：

* 评价 reliability；
* 统计 Gaussian 真实误差；
* 计算最终几何指标。

不能用于：

* 选择被回收 Gaussian；
* 生成候选位置；
* 调整测试场景参数；
* 训练阶段的实际决策。

---

## 7.2 Gaussian 几何误差标签

最基础标签：

[
d_i=
d(\mu_i,\mathcal M_{GT})
]

归一化：

[
\hat d_i=
\frac{d_i}{D_{\text{object}}}
]

同时增加尺度归一化距离：

[
d_i^{scale}
===========

\frac{
d(\mu_i,\mathcal M_{GT})
}{
s_{i,n}+\epsilon
}
]

其中 (s_{i,n}) 是 Gaussian 沿表面法向的有效尺度。

建议同时评价：

* 连续误差 (\hat d_i)；
* 二分类错误标签；
* Gaussian 法线与 GT 法线夹角。

错误标签初始定义：

```text
correct:
d_norm < 0.005

ambiguous:
0.005 ≤ d_norm < 0.02

wrong:
d_norm ≥ 0.02
```

最终阈值要根据 TransLab 坐标单位和物体直径确认。

---

## 7.3 评价指标

对每个单项特征和组合风险报告：

* AUROC；
* AUPRC；
* Spearman 相关系数；
* top-5%、10%、20% 风险组平均 mesh distance；
* top-risk 组错误率；
* 风险分位数—真实误差曲线；
* bootstrap 95% 置信区间。

---

## 7.4 Gate 1

满足以下条件才能进入删除阶段：

1. AUPRC 明显高于错误类别的随机比例；
2. top-10% 风险组错误率至少是整体的 1.5 倍；
3. 高风险组 mesh distance 显著高于低风险组；
4. scene_01 和 scene_03 趋势一致；
5. 至少两个特征具有稳定的跨场景相关性。

AUROC 0.70、Spearman 0.30 可以作为参考，但不是硬性论文标准。

### Gate 1 失败处理

按顺序调整：

1. 检查相机、坐标系和投影是否正确；
2. 去掉 visibility variance；
3. 降低 mask 权重；
4. 提高 normal 和 surface-support 权重；
5. 将背景漂浮点和内部错误点分开建模；
6. 仍无效则暂停 recycling，不进入后续阶段。

---

# 八、Stage 2：Gaussian 状态分类

## 8.1 Retain

满足：

* 风险低；
* 法线稳定；
* 表面邻域支持高；
* 具有稳定贡献。

操作：

> 完全保留。

---

## 8.2 Recycle

第一版只选择高置信度目标。

### 类型 A：背景漂浮

```text
mask support低
+ 风险高
+ contribution低或中
```

### 类型 B：内部孤立

```text
mask support高
+ surface support低
+ depth-order conflict高
+ contribution低或中
```

### 类型 C：异常大椭球

```text
scale异常
+ normal conflict高
+ surface support低
```

---

## 8.3 Ambiguous

满足：

```text
风险高
+ contribution高
```

或：

```text
mask support高
+ normal/depth冲突高
+ 位于薄边和杯口区域
```

这些 Gaussian 暂时不删除。

可选操作：

* 降低位置学习率；
* 保留到后续全局优化；
* 允许普通 split；
* 只做可视化分析。

---

# 九、Stage 2A：Prune-only 实验

在重播种前，必须证明“会删”。

## 9.1 对比组

保持删除比例相同：

1. Random prune；
2. Low-opacity prune；
3. Low-contribution prune；
4. Gradient-based prune；
5. Reliability-guided prune；
6. GT mesh-distance oracle prune，仅作上界分析。

---

## 9.2 删除比例

第一轮：

```text
2%
5%
10%
```

不要第一轮测试 15%，透明场景中风险较大。

默认使用 5%。

---

## 9.3 评价时机

每种方法评价两次：

### 立即评价

删除后不训练：

* PSNR；
* Chamfer；
* F-score；
* 被删除 Gaussian 的真实 mesh distance；
* 漂浮比例。

### 短优化评价

统一优化 500 步后再次评价。

所有方法：

* 使用相同学习率；
* 使用相同迭代数；
* 不新增 Gaussian。

---

## 9.4 Gate 2

Reliability prune 至少满足：

* 同删除比例下 PSNR 损失小于 random；
* 被删除 Gaussian 的真实误差高于 opacity 和 contribution；
* 几何指标不明显恶化；
* 500 步恢复后接近或优于原 checkpoint；
* 两个开发场景趋势一致。

### Gate 2 失败处理

1. 删除比例从 5% 降到 2%；
2. 只删除 high-risk + low-contribution；
3. 不删除杯口和透明边缘；
4. 对不同错误类型设置独立规则；
5. 若仍失败，回到 Stage 1。

---

# 十、Stage 3：几何欠覆盖检测

欠覆盖不能使用“最终渲染 alpha 低”直接判断。

应该定义为：

> 某个透明 mask 内像素缺少与参考表面深度和法线相符的几何 Gaussian 支持。

---

## 10.1 几何表面支持图

对视角 (v) 的像素 (p)，定义：

[
S_v^{geo}(p)
============

\sum_{i\in\mathcal G(p)}
w_{iv}^{geo}
\cdot
\exp
\left(
-\frac{|z_{iv}-D_v^{ref}(p)|}{\sigma_d}
\right)
\cdot
C_{iv}^{normal}
]

其中：

[
C_{iv}^{normal}
===============

\max
\left(
0,
\left|
n_{iv}^{\top}N_v^{prior}(p)
\right|
\right)
]

这里的 (w_{iv}^{geo}) 应优先使用：

* 几何阶段贡献；
* depth-window 内贡献；
* 表面支持权重。

不要直接使用最终外观 alpha。

---

## 10.2 欠覆盖分数

[
H_v(p)
======

M_v(p)
\left[
\alpha
\left(
1-\widetilde S_v^{geo}(p)
\right)
+
\beta E_v^{normal}(p)
+
\gamma E_v^{depth}(p)
+
\delta E_v^{edge}(p)
\right]
]

其中：

### 几何支持不足

[
1-\widetilde S_v^{geo}(p)
]

### 法线残差

[
E_v^{normal}(p)
===============

1-
\left|
\hat N_v(p)^\top N_v^{prior}(p)
\right|
]

### 深度局部异常

[
E_v^{depth}(p)
==============

\operatorname{Median}_{q\in\mathcal N(p)}
\left|
D_v^{ref}(q)-D_v^{ref}(p)
\right|
]

### 边缘加权

[
E_v^{edge}(p)
=============

\exp
\left(
-\frac{\operatorname{dist}(p,\partial M_v)^2}
{2\sigma^2}
\right)
]

边缘项只能作为辅助，不能让所有候选都集中在 mask 轮廓。

---

## 10.3 候选像素选择

每个视角：

1. 仅保留透明 mask 内像素；
2. 取欠覆盖分数 top 5%～10%；
3. 执行 2D NMS；
4. 每个视角最多选固定数量；
5. 避免大量候选来自同一局部区域。

---

# 十一、Stage 3A：3D 候选生成与跨视角验证

## 11.1 生成视图与验证视图分离

这是避免循环自证的关键。

对源视角 (v_s) 的候选像素 (p)：

[
x=
\pi_{v_s}^{-1}
\left(
p,D_{v_s}^{ref}(p)
\right)
]

源视图只负责生成候选，不能同时负责证明候选正确。

---

## 11.2 在其他视图中验证

将候选点 (x) 投影到其他视图 (v\neq v_s)。

### Mask 验证

[
M_v(\pi_v(x))=1
]

### 图像边界验证

投影必须在有效图像范围内。

### 深度验证

[
e_v^{depth}(x)
==============

\frac{
|z_v(x)-D_v^{ref}(\pi_v(x))|
}{
D_v^{ref}(\pi_v(x))+\epsilon
}
]

### 法线验证

候选法线 (n_x) 与其他视图法线先验一致：

[
e_v^{normal}(x)
===============

1-
\left|
n_{x,v}^{\top}
N_v^{prior}(\pi_v(x))
\right|
]

### 前方遮挡验证

如果候选明显位于参考第一表面之后，则该视图不提供正支持。

---

## 11.3 候选支持条件

第一版建议：

```text
除源视图外支持视图数 ≥ 3
mask支持率 ≥ 0.70
深度一致视图比例 ≥ 0.60
法线一致视图比例 ≥ 0.60
```

视图数量较少时，使用比例阈值代替绝对数量。

---

## 11.4 候选评分

[
Q(x)
====

\lambda_h Q^{undercoverage}
+
\lambda_m Q^{mask}
+
\lambda_d Q^{depth}
+
\lambda_n Q^{normal}
+
\lambda_r Q^{repulsion}
]

其中 repulsion：

[
Q^{repulsion}(x)
================

\min_j
\frac{
|x-\mu_j|
}{
s_j+\epsilon
}
]

候选不能与已有可信 Gaussian 完全重合。

---

## 11.5 3D NMS

所有候选按 (Q(x)) 降序排列：

```text
selected = []

for candidate in sorted_candidates:
    if candidate距离已有selected点 > r_nms:
        selected.append(candidate)

    if len(selected) == K:
        break
```

其中：

[
K=N_{\text{removed}}
]

如果有效候选不足 (K)，不要使用低质量候选凑数。

此时：

* 减少回收数量；
* 或只重生有效候选数；
* 同时从删除集合中恢复未使用预算对应的 Gaussian。

最终仍保持预算一致。

# 十二、Stage 3B：新 Gaussian 初始化

## 12.1 位置

使用多视角验证后的候选位置。

---

## 12.2 法线与旋转

候选法线来源优先级：

1. 多视角法线先验融合；
2. 邻近可信 Gaussian 法线；
3. 局部平面拟合。

融合：

[
n_{\text{new}}
==============

\operatorname{Normalize}
\left(
\sum_v w_v n_v
\right)
]

rotation 设置为：

> Gaussian 最短轴与 (n_{\text{new}}) 对齐。

---

## 12.3 Scale

使用邻近可信 Gaussian 的中位数：

[
s_t=
\operatorname{Median}*{j\in\mathcal N}
s*{j,t}
]

切向尺度：

[
s_{t1}=s_{t2}=s_t
]

法向尺度：

[
s_n=\rho s_t
]

初始：

```text
ρ = 0.3
```

消融：

```text
ρ ∈ {0.2, 0.3, 0.5}
```

---

## 12.4 Opacity

新点 opacity 保守初始化：

```text
new_opacity =
0.5 × 邻域可信Gaussian opacity中位数
```

或 sigmoid 前：

```text
-2.0
```

不要让新 Gaussian 一出生就产生强遮挡。

如果 TSGS 区分几何和外观 opacity：

* geometry opacity 使用中等初值；
* appearance opacity 使用更低初值；
* 两者分阶段优化。

---

## 12.5 颜色与 SH

第一版：

* DC 颜色使用 support views 的鲁棒中位数；
* 高阶 SH 全部初始化为 0。

也可以复制最近可信 Gaussian 的 DC 项，但不能直接复制全部高阶 SH。

---

# 十三、Stage 4：局部精修

局部精修应替代一部分正常训练步骤，而不是额外增加总迭代数。

---

## 13.1 Support views

每个新 Gaussian 选择 4～8 个 support views，依据：

* mask 支持；
* 深度一致性；
* 法线一致性；
* 有效投影面积；
* 候选在图像内；
* 不被明显遮挡。

---

## 13.2 局部优化区域

优化对象：

* 新 Gaussian；
* 距离新 Gaussian 最近的局部旧 Gaussian；
* 与新 Gaussian 投影区域重叠的旧 Gaussian。

冻结：

* 远离重生区域的旧 Gaussian；
* 相机参数；
* 全局曝光参数。

---

## 13.3 Phase A：几何稳定

建议 300～500 步。

优化：

* position；
* rotation；
* scale；
* geometry opacity。

冻结：

* 高阶 SH；
* 远处旧 Gaussian；
* appearance opacity。

损失：

[
\mathcal L_A
============

\lambda_{rgb}\mathcal L_{rgb}
+
\lambda_n\mathcal L_{normal}
+
\lambda_d\mathcal L_{depth}
+
\lambda_f\mathcal L_{flat}
+
\lambda_r\mathcal L_{rep}
]

初始权重可设置为：

```yaml
rgb: 1.0
normal: 0.5
depth: 0.25
flat: 0.05
repulsion: 0.01
```

这些只是第一轮工程值，只能在开发场景上调整。

---

## 13.4 Phase B：局部外观恢复

建议 500～1000 步。

优化：

* 新 Gaussian 全部参数；
* 邻域旧 Gaussian 低学习率；
* DC 和 SH；
* appearance opacity。

新点学习率高于旧点，例如：

```text
lr_new = baseline_lr
lr_neighbor_old = 0.1 × baseline_lr
```

---

## 13.5 Phase C：全局继续训练

恢复 TSGS 正常训练。

建议：

* 降低新 Gaussian 的位置学习率；
* 停止或严格约束标准 densification；
* 保证 Gaussian 总量基本不变；
* 保留正常的颜色和 opacity 优化。

---

# 十八、阶段 Gate

## Gate 1：可靠性有效

通过条件：

* 风险与真实误差显著相关；
* top-risk 组错误率明显高于整体；
* AUPRC 高于随机水平；
* 两个开发场景趋势一致。

失败：

> 停止后续 recycling，先修复 reliability。

---

## Gate 2：删除安全

通过条件：

* reliability prune 优于 random、opacity、contribution；
* 相同删除比例下渲染损失更小；
* 被删除 Gaussian 真实误差更高；
* 短优化后几何不恶化。

失败：

> 降低删除比例，并加强 contribution gate。

---

## Gate 3：重播种有效

通过条件：

* prune + reseed 优于 prune-only；
* random reseed 明显弱于跨视角验证 reseed；
* 新 Gaussian 更接近 GT 表面；
* surface proximity 提高；
* floating ratio 降低。

工程目标：

```text
平均Chamfer相对下降约5%
或
F-score提升约2个百分点
```

这是项目决策参考，不是论文必须达到的固定门槛。

---

## Gate 4：局部精修有效

要求：

```text
reseed + local refinement
优于
reseed without refinement
```

至少在：

* 几何；
* normal；
* RGB；

中的两项有稳定收益。

---

## Gate 5：全数据稳定

建议达到：

* 8 个场景中至少 6 个几何提升；
* 不超过 1 个场景明显退化；
* 平均几何指标优于 TSGS；
* Gaussian 数量差异低于 1%；
* 总训练成本可接受。

---

# 二十二、论文最终贡献

最终论文可以形成三项贡献。

## Contribution 1：透明几何可靠性

提出跨视角透明几何可靠性评估，区分：

* 对 RGB 有贡献；
* 与真实表面几何可信；

这两个并不等价的概念。

---

## Contribution 2：几何预算重分配

将高风险、非关键 Gaussian 从：

* 折射背景；
* 透明物体内部；
* 错误深度层；

回收到经过多视角验证的欠覆盖透明表面。

固定预算是控制变量和效率约束。

---

## Contribution 3：几何—外观分阶段修复

新 Gaussian 先恢复：

* 位置；
* 法线；
* 尺度；
* 几何表面支持；

再恢复：

* opacity；
* 颜色；
* SH 外观。

避免新点再次被纯 photometric loss 拉向折射背景。

# 二十三、最终成功标准

这个项目是否成立，主要看四件事：

[
\boxed{
\text{Risk能够识别几何错误}
}
]

[
\boxed{
\text{Reliability prune优于简单剪枝}
}
]

[
\boxed{
\text{跨视角重播种优于随机重播种}
}
]

[
\boxed{
N_{\text{ours}}\approx N_{\text{TSGS}},
\quad
CD_{\text{ours}}<CD_{\text{TSGS}},
\quad
F_{\text{ours}}>F_{\text{TSGS}}
}
]

最关键的执行原则是：

> **先完成 Gate 1，再写删除代码；先完成 Gate 2，再实现复杂重播种；在开发场景上锁定参数后，禁止为其他场景单独调参。**

这样即使后面的完整 recycling 效果不理想，你也能准确知道失败发生在“检测、删除、候选位置还是优化”，而不会再次陷入所有模块一起运行、指标整体恶化却无法定位原因的情况。
